SLA Fallback Automático

Quando habilitado, se o modelo principal demorar mais que o timeout configurado para responder (TTFT), automaticamente troca para um modelo mais rápido. DESABILITADO por padrão - habilite apenas se souber o que está fazendo.

Habilitar pode resultar em respostas de menor qualidade

Comportamento: Se o modelo principal não responder em X segundos, o sistema automaticamente tenta com o modelo de fallback. Útil para alta disponibilidade, mas pode comprometer qualidade em tarefas complexas.
